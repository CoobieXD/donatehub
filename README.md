# DonateHub

Страничка-хаб для донатов на стриме. Зритель заходит — по IP определяется его регион, и ему показываются только те платформы, которые работают в его стране. Украинцу — Monobank и Donatello, россиянину — DonationAlerts и DonatePay, остальным — Streamlabs, Ko-fi и т.д.

Чистый HTML/CSS/JS, никаких фреймворков и сборки. Закинул на GitHub Pages — и работает.

## Возможности

- Гео-детекция по IP с fallback API
- Переключатель региона/языка (Українська / Русский / English)
- Тёмная и светлая тема (подхватывает системную, переключается вручную)
- Авто-редирект на любимую платформу (трекинг кликов в localStorage)
- QR-коды для донат-ссылок и крипто-адресов (hover на десктопе)
- Копирование реквизитов в буфер обмена
- Адаптивная вёрстка (мобилка / планшет / десктоп)
- Fade-in анимации при загрузке
- Валидация конфига с экраном ошибки
- Real-time уведомления через ntfy.sh + live dashboard

## Как использовать

1. Форкни или склонируй репозиторий
2. Открой `config.json` — это единственное, что нужно менять:
   - `streamerName` — твой ник
   - `accentColor` — цвет кнопок (по умолчанию Twitch-фиолетовый)
   - `platforms` — вставь свои ссылки вместо `XXXXX`, лишние платформы выключи (`"enabled": false`)
   - `subscriptions` — подписки (Patreon, Boosty, Ko-fi), выключены по умолчанию
   - `requisites` — крипто-адреса и карты для прямого перевода
   - `ogImage` — URL картинки для превью при шаринге ссылки
3. Переведи описания платформ на язык региона (UA — українською, RU — на русском, _global — in English)
4. В `index.html` есть `<noscript>` блок с захардкоженными ссылками — обнови их тоже
5. (Опционально) Настрой уведомления — см. секцию «Уведомления» ниже
6. Залей на GitHub Pages или просто открой через HTTP-сервер

## Как работает

Зритель открывает страницу → запрос к `ipapi.co` определяет страну (fallback: `ip-api.com`) → страна маппится на регион (`UA`, `RU`, `_global`) → показываются платформы для этого региона, отсортированные по приоритету → интерфейс переводится на язык региона.

Если зритель вручную выбрал регион через глобус — выбор сохраняется в `localStorage` и используется при следующих визитах вместо гео-детекции.

Если для региона нашлась только одна платформа — идёт авто-редирект с отсчётом. Если ни одной — fallback на `_global`. Если зритель несколько раз кликал по одной платформе — при следующем визите сразу предложит её с возможностью отмены.

Регион кешируется в `sessionStorage`, так что повторные заходы не дёргают API.

## Уведомления (ntfy.sh)

DonateHub может отправлять real-time уведомления когда зритель открывает страницу, кликает по платформе или копирует реквизит. Используется бесплатный [ntfy.sh](https://ntfy.sh) — никакой инфраструктуры не нужно.

### Настройка

1. Придумай уникальный topic (например `donatehub-xyz123abc`) — это твой «канал»
2. В `config.json` включи уведомления:
   ```json
   "notifications": {
     "enabled": true,
     "ntfyTopic": "donatehub-xyz123abc"
   }
   ```
3. Открой `dashboard.html` в браузере, введи тот же topic и нажми Connect

### Dashboard

`dashboard.html` — standalone страница для мониторинга событий:

- Live-лента событий (новые сверху)
- Счётчики: просмотры / клики / копирования за сессию
- Звуковые уведомления (опционально)
- Topic сохраняется в localStorage

### Типы событий

| Тип | Когда | Данные |
|-----|-------|--------|
| `page_view` | Зритель открыл страницу | region, lang |
| `platform_click` | Клик по платформе (секция «Донат») | platformId, platformName |
| `requisite_copy` | Копирование реквизита | requisiteId, requisiteName |

> **Важно:** topic — публичный идентификатор. Используй длинную случайную строку, чтобы никто посторонний не мог подписаться на твои события.

## Настройка платформ

Каждая платформа в `config.json`:

| Поле          | Что это                                             |
|---------------|-----------------------------------------------------|
| `enabled`     | `true/false` — показывать или нет                   |
| `id`          | Уникальный ключ (по нему подтягивается SVG-иконка)  |
| `name`        | Название на карточке                                 |
| `url`         | Куда ведёт кнопка                                    |
| `description` | Подпись под названием (на языке региона)              |
| `priority`    | Порядок (1 — первый)                                 |

Платформы группируются по регионам: `"UA"`, `"RU"`, `"_global"`. Если один и тот же сервис нужен в нескольких регионах — заведи отдельные записи с разными `id` (например `donationalerts-ua` и `donationalerts-global`).

### Доступные id для иконок

`monobank`, `donatello`, `donationalerts`, `donationalerts-ua`, `donationalerts-global`, `donatepay`, `streamlabs`, `kofi`, `paypal`, `patreon`, `boosty`, `steam`, `usdt`, `btc`, `ton`, `mastercard`, `visa`

## Тестирование

Для локальной разработки нужен HTTP-сервер (`npx serve`), потому что `config.json` загружается через `fetch`.

Чтобы проверить, как выглядит страница для разных регионов, добавь `?region=` в URL:

```
?region=UA   — Украина (українською)
?region=RU   — Россия/СНГ (на русском)
?region=DE   — всё остальное (_global, English)
```

```

style.css       — тёмная/светлая тема, CSS variables, адаптив (480px, 768px)
script.js       — логика: гео-детекция, i18n, роутинг, рендеринг, тема, уведомления
dashboard.html  — live dashboard для просмотра событий через ntfy.sh SSE
favicon.svg     — фавиконка
```
